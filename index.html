git init: Inicializa un nuevo repositorio Git en el directorio actual.

 

 

git init

git clone: Clona un repositorio Git existente en un nuevo directorio.

 

 

git clone https://ejemplo.com/repo.git

git add: Agrega cambios al área de preparación (staging) para ser incluidos en el próximo commit.

 

 

git add archivo.txt

git commit: Crea un nuevo commit con los cambios agregados al área de preparación.

 

 

git commit -m "Mensaje del commit"

git status: Muestra el estado actual del repositorio, incluyendo los archivos modificados, agregados o eliminados.

 

 

git status

git push: Sube los commits locales al repositorio remoto.

 

 

git push origin master

git pull: Obtiene los cambios del repositorio remoto y los fusiona con la rama local.

 

 

git pull origin master

git branch: Lista las ramas en el repositorio, o crea una nueva rama.

 

 

git branch

git branch nueva_rama

git checkout: Cambia de rama o restaura archivos desde un commit específico.

 

 

git checkout mi_rama

git checkout HEAD~1 archivo.txt

git merge: Fusiona una rama con otra.

 

 

git merge otra_rama

git log: Muestra el historial de commits.

 

 

git log

git reset: Permite deshacer cambios en el área de preparación o en el directorio de trabajo.

 

 

git reset HEAD archivo.txt

git remote: Muestra los repositorios remotos configurados.

 

 

git remote -v

git config: Configura opciones de Git como el nombre de usuario, el correo electrónico, etc.

 

 

git config --global user.name "Tu Nombre"

git config --global user.email "tu@email.com"

Encapsulación:

Los atributos de una clase pueden ser protegidos de acceso externo usando los modificadores de acceso (private, protected, public). Private (privado):

Los miembros (atributos o métodos) de una clase declarados como private solo son accesibles dentro de la misma clase.

No son visibles ni accesibles desde fuera de la clase, ni siquiera por las subclases de la misma.

Se utiliza para encapsular la implementación de la clase y evitar que se modifique accidentalmente desde fuera.

Protected (protegido):

Los miembros declarados como protected son accesibles dentro de la misma clase, en las subclases (herencia) y en otras clases dentro del mismo paquete.

No son accesibles desde clases fuera del paquete a menos que también sean subclases.

Se utiliza para permitir el acceso controlado a miembros de una clase desde las subclases y otras clases dentro del mismo paquete.

Public (público):

Los miembros declarados como public son accesibles desde cualquier lugar.

No hay restricciones en el acceso a los miembros públicos.

Se utiliza para permitir un acceso completo a los miembros desde cualquier parte del programa.

class MyException extends Exception {

         super(message);

    }

}

Defino la clase  MyException que extiende la clase Exception. Esto significa que MyException es una excepción comprobada, lo que implica que cualquier método que lance MyException debe manejarla de alguna manera como hacerlo dentro de un bloque try-catch.

 

private static void throwCustomException() throws MyException {

    throw new MyException("se genera un error");

}

Este método  throwCustomException está diseñado para lanzar una excepción MyException. Aquí se crea una instancia de MyException y se lanza con la instrucción throw.
